# I18n / Translation

Zero Framework includes a lightweight i18n layer with file-based translations, inline translation blocks, and template helpers.

## Quick Start

```php
// resources/views/pages/home.php
<h1>{{ __('welcome.title') }}</h1>
<p>@t('welcome.message', ['name' => $name])</p>
```

Create translation files:

```text
resources/i18n/en/pages/home.json
resources/i18n/id/pages/home.yaml
```

Example JSON:

```json
{
  "welcome": {
    "title": "Welcome",
    "message": "Hi {{ $name }}"
  }
}
```

Example YAML:

```yaml
welcome:
  title: Welcome
  message: Hi {{ $name }}
```

## Translation Helpers

- `__(string $key, array $replacements = [], ?string $context = null, ?string $locale = null): string`
- `@t('key', ['name' => $name])` (template alias for `__()`)

Missing keys return the original key string.

## Inline Translation Blocks

Inline blocks can be used in any template (including child components):

```php
@i18n('yaml')
en:
  notice_form: If you {{ $name }} did not request a password reset, ignore this email.
  info:
    message: Just example recursive
id:
  notice_form: Jika {{ $name }} tidak meminta reset password, abaikan email ini.
@endi18n

<p>@t('notice_form', ['name' => $name])</p>
<p>@t('info.message')</p>
```

Inline YAML blocks also accept JSON payloads for convenience.

## External Translation Files

By default the view name is used to locate translations:

```php
// resources/views/pages/profile.php
<p>{{ __('profile.title') }}</p>
```

Will load (based on locale):

```text
resources/i18n/{locale}/pages/profile.json
resources/i18n/{locale}/pages/profile.yaml
resources/i18n/{locale}/pages/profile.yml
```

You can explicitly load a file:

```php
@i18n('emails/reset-password')
```

## Locale Resolution

Locale resolution is configured in `config/i18n.php` and supports multiple strategies in order:

1) URL prefix (e.g. `/en/{page}`)
2) domain / subdomain (host-based) mapping (e.g. `id.zero.local` or `id.example.com`)
4) session
5) cookie
6) header (default `X-Locale`)
7) custom resolver callback

The first supported locale found is used. If none match, the default locale is used. To use host-based locales, set `resolver` to `domain` and define a host map. Subdomain fallback is enabled by default (offset `0`, ignore `['www']`) unless explicitly disabled.

### Fallbacks

Fallback locales are tried in order when a key is missing:

```
'en-GB' -> 'en' -> default
```

Configure fallbacks via `fallback_locales`.

## Configuration

### `config/i18n.php`

```php
return [
    'web' => [
        'default_locale' => 'en',
        'supported_locales' => ['en', 'id', 'it', 'cn'],
        'fallback_locales' => ['en'],
        'resolver' => 'url',
        'url' => [
            'enabled' => true,
            'segment' => 1,
        ],
        'session_key' => 'locale',
        'cookie_key' => 'locale',
        'header' => 'X-Locale',
        'domain' => [
            'map' => [
                'id.zero.local' => 'id',
                'nl.zero.local' => 'nl',
                'it.zero.local' => 'it',
                'cn.zero.local' => 'cn',
            ],
        ],
        'custom_resolver' => null,
        'path' => 'resources/i18n',
    ],
    'email' => [
        'default_locale' => 'en',
        'supported_locales' => ['en', 'id', 'it', 'cn'],
        'fallback_locales' => ['en'],
        'resolver' => 'custom',
        'custom_resolver' => null,
        'domain' => [
            'map' => [
                'id.zero.local' => 'id',
                'nl.zero.local' => 'nl',
                'it.zero.local' => 'it',
                'cn.zero.local' => 'cn',
            ],
        ],
        'path' => 'resources/i18n',
        'view_prefix' => 'mail',
        'context' => 'mail',
        'subdomain' => [
            'offset' => 0,
            'ignore' => ['www'],
        ],
    ],
];
```

## Notes

- YAML parsing uses the PHP `yaml` extension if installed; otherwise a minimal YAML parser is used.
- No caching is enabled for translations yet.

## Helpers

Zero ships with convenience helpers for reading and switching locales:

```php
locale();              // Current active locale for the context
locales();             // Configured locale list (supported_locales or default)
set_locale('id');      // Apply locale using config rules
```

`set_locale()` validates against `supported_locales` and falls back to `default_locale` when needed. It only persists to session/cookie when the resolver is set to `session` or `cookie`.
